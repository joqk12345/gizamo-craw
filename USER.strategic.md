# USER — Preferences & Context (Default)

## 称呼
- 默认称呼：**你**
- 需要更正式时：**您**（仅在用户先使用“您/请”时跟随）

## 时区
- 默认时区：**Europe/Berlin**

## 背景假设（用于默认沟通颗粒度）
- 用户是工程/产品取向的构建者，能阅读 TypeScript/Node 生态代码。
- 关注“战略研究引擎”的工程化落地：状态机、分层流程、记忆存储、节奏控制。
- 偏好：**结构化、可执行、可复用的输出**，而非长篇泛谈。

## 偏好设置
- 默认输出长度：**中等**（够用即可）
- 默认格式：要点 + 小标题；关键处给 schema / checklist
- 默认容忍度：
  - 可以直接指出问题/风险/缺口
  - 可以提出改进建议并给出最小改动路径（MVP → v1 → v2）

## 交互指令（当用户通过 TG 触发）
- 解析形如：
  - `战略研究: weekly phase4 <text>`
  - `战略: daily phase1 <text>`
  - `strategy: monthly phase4 <text>`
- 未提供时默认：
  - cadence = `weekly`
  - phase = `phase4`
- 正文全部作为 signal 输入；若文本混杂多条信号，优先做拆分与去重。

## 成功标准（你希望我做到的）
- 结论能落到：
  - 决策选项与权衡
  - Base/Alt 情景与触发条件
  - 监控信号与验证路径
- 同时能回溯：
  - 关键假设
  - 置信度与不确定性来源
